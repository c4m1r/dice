Сделай Онлайн приложение “Dice” для броска DnD и прочих костей в режимах 2D и 3D. Центральная область — “стол” на весь экран (canvas). Два боковых меню: слева управление пулом костей и режимами, справа — история и “кинуть одну” с силой по удержанию.

ОБЯЗАТЕЛЬНЫЙ ТЕХСТЕК (не отклоняйся)
- Vite + React + TypeScript
- 3D: three.js (БЕЗ react-three-fiber)
- Физика: cannon-es
- Состояние: zustand (или чистый reducer, но предпочтительно zustand)
- Хранилище: localStorage (история + настройки)
- Стили: CSS Modules (или Tailwind — выбери одно и придерживайся)
- Никаких бекендов, всё на фронте.

ПРИОРИТЕТ: МОБИЛЬНАЯ ПРОИЗВОДИТЕЛЬНОСТЬ
- Цель: стабильно работать на телефонах (mid-range Android/iPhone).
- Упор на низкополигональную геометрию и минимум draw calls:
  * Используй простые THREE геометрии (TetrahedronGeometry / BoxGeometry / OctahedronGeometry / DodecahedronGeometry / IcosahedronGeometry) с небольшим detail=0.
  * ОДИН общий материал на тип кости (MeshStandardMaterial или MeshLambertMaterial), без текстур.
  * Включи pixelRatio = min(devicePixelRatio, 1.5) на мобильных.
  * Ограничь максимальное число костей на столе по умолчанию до 12 (настройка можно увеличить).
  * Упрощённый свет: 1 HemisphereLight + 1 DirectionalLight.
  * Выключи/не используй тяжелые postprocessing эффекты.
- Используй requestAnimationFrame, но при отсутствии движения физики переходи на “idle mode”:
  * если мир “успокоился” — перестань рендерить каждый кадр (рендер только по событиям), либо рендери с пониженной частотой.
- При “prefers-reduced-motion” автоматически включай 2D и сниженные анимации.

ГЛАВНЫЙ ЛЭЙАУТ (1:1)
Экран: 3 зоны
A) Центр: “TABLE” — canvas на весь доступный экран между сайдбарами. В 3D тут кости с физикой, в 2D — упрощённая визуализация/анимация + раскладка результатов.
B) Левый сайдбар (Control Panel):
  - Самый верх: кнопка “☰ Меню” (overlay/drawer) с пунктами:
    * Настройки
    * Энциклопедия
    * О приложении
  - Блок “Кости”: список типов (минимум d4,d6,d8,d10,d12,d20). Для каждой строки:
    * метка (d20)
    * короткое описание (tooltip/иконка i)
    * количество: [-] [число] [+] (0..20, но предупреждай/ограничивай по общему лимиту)
  - Переключатели:
    * 2D / 3D
    * “Броски” / “Гадания”
  - Modifier: поле -10..+10
  - Внизу: большая красная кнопка “БРОСИТЬ” — бросает весь пул.
C) Правый сайдбар (History Panel):
  - Вверху: маленькая красная кнопка “Кинуть одну”
    * удержание нажатия накапливает силу (визуальный индикатор)
    * отпускание — бросок 1 кости с силой от удержания
    * кидает последнюю выбранную кость; если пул пуст — d20
  - Ниже: история бросков (scroll), карточками:
    * время
    * режим (Броски/Гадания) + 2D/3D
    * состав пула
    * результаты по каждой кости + сумма + modifier
    * для “Гаданий” — текст интерпретации
    * кнопки: “Повторить”, “Копировать”

MOBILE UX (обязательно)
- На ширине < 900px сайдбары превращаются в выезжающие панели (drawers):
  * слева — Control Drawer, справа — History Drawer
  * в центре сверху показывай две кнопки/иконки для открытия левого/правого drawer.
- Кнопки должны быть крупные, отступы удобные для пальца.
- Используй pointer events (pointerdown/up/move), не mouse-only.

КЛЮЧЕВЫЕ ФИЧИ (обязательны)
1) 3D режим (low poly)
- Реальная физика: кости падают на плоскость-стол, сталкиваются, трение/упругость.
- Настройки физики:
  * Throw Force (5..50)
  * Spin Force (0..50)
- Определение результата после “settled”:
  * критерий: линейная скорость и угловая скорость ниже порога (например 0.15) в течение N кадров (например 20).
  * определить “верхнюю грань” по ориентации:
    - Для каждого типа кости задай таблицу локальных нормалей граней и значений.
    - Преобразуй нормали в мировые координаты и выбери максимальный dot(worldUp).
  * Если для сложных платоновых тел это слишком трудоёмко — ДОПУСТИМО сделать “стабильный режим результата”: RNG заранее выбирает число, а визуализация — просто физическая анимация без вычисления top-face; но:
    - В истории/результатах число должно быть корректным и равномерно случайным.
    - В настройках должен быть тумблер:
      * “Результат по физике” (экспериментально, может быть медленнее)
      * “Результат по RNG” (по умолчанию, быстрый и стабильный)
- Камера: минимальный OrbitControls, но на мобильном ограничь (без панорамирования, только лёгкий rotate + pinch zoom) и выставь damping.

2) 2D режим
- Canvas2D: анимация броска (быстрый shake/rotate), затем показ значений.
- Логика пула и история полностью общие с 3D.

3) “Броски” vs “Гадания”
- “Броски”: обычный dice roller.
- “Гадания”: подрежимы:
  A) “На одной кости (4 броска чёт/нечёт)” → 4 броска, X=чёт, O=нечёт, трактовка по таблице.
  B) “На двух костях” → 2 кости, трактовка по сумме (и показать комбинацию).
  C) “На трёх костях” → 3 кости, трактовка по сумме.
- Таблицы интерпретаций — в коде (RU), легко расширяемые.
- Каждое гадание пишет interpretationText в историю.

4) История
- 200 последних событий, localStorage: dice_table_history, dice_table_settings.
- Очистка истории в настройках.

5) “Кинуть одну” с удержанием
- pointerdown → старт, pointerup → вычисли силу:
  power = clamp(holdMs / 1500, 0, 1)
  throwApplied = baseThrow * lerp(0.4, 1.6, power)
  spinApplied  = baseSpin  * lerp(0.2, 1.8, power)
- Записать событие single-roll в историю.

ТЕКСТЫ ДЛЯ ЭНЦИКЛОПЕДИИ (встроить в “☰ Меню → Энциклопедия”)
Страница “Кости” — улучшенные описания:
- d4 — четырёхгранник (тетраэдр): компактная кость для небольших значений; результат читается по верхней зоне/вершине; шутливо “самая опасная”.
- d6 — шестигранник (куб): универсальная кость; из-за неё все кости часто зовут “кубиками”.
- d8 — восьмигранник (октаэдр): для эффектов средней силы; результат по верхней грани.
- d10 — десятигранник: 1–10 или 0–9; две d10 часто как d100; 0 при одиночном броске обычно = 10.
- d12 — двенадцатигранник (додекаэдр): широкий диапазон без хаоса d20.
- d20 — двадцатигранник (икосаэдр): равномерное распределение, критические исходы встречаются чаще.

Страница “Гадания” — вставь таблицы трактовок (перенеси полностью из текста пользователя):
- 1 кость (XXXX ... OOOO)
- 2 кости (суммы 2..12)
- 3 кости (суммы 3..18)
